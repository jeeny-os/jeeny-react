# JeenyForm

<div className="explainer">
  A headless wrapper that helps you create type-checked forms that work with the
  Jeeny API.
</div>

<div className="tip-wrapper">
  The JeenyForm component is a headless component. It lets you take care of
  design and user experience while providing a typesafe way to easily access the
  Jeeny API.
</div>

<div className="tip-wrapper">
  The JeenyForm component gives you easy access to every mutation in the Jeeny
  API. Through it's <code>renderForm</code> prop you receive the return values
  of a fully typed{" "}
  <a href="https://react-hook-form.com/get-started" target="_blank">
    React Hook Form
  </a>{" "}
  <code>useForm</code> hook. The typing comes from the value that you enter into
  the <code>action</code> prop.
</div>

<div className="tip-wrapper">
  The JeenyForm component takes four props. The first prop, <code>action</code>,
  indicates what GraphQL mutation you wish to interact with. Depending on that
  prop, whether it is <code>createItem</code> or <code>pickLocation</code>, the
  expectations of the <code>defaultValues</code> and <code>renderForm</code>{" "}
  props will change. In the code example below, those two props expect to see
  the Jeeny Type <code>CreateItemInput</code>.
</div>

<div className="tip-wrapper">
  If we were to change the <code>action</code> to <code>saveSupplier</code> then
  those two props would then expect to use the Jeeny Type
  <code>SupplierInputUpdate</code>.
</div>

<div className="tip-wrapper warning-box">
  Unfortunately, when using the <code>register</code> function of React Hook
  Form we can't properly typecheck a nested form field. In the example below you
  will see that in the last input field, we need to set the parameter of the
  register function to <code>any</code>. We are on the look out for a fix.
</div>
